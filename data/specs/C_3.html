프로젝트 개요

광고 문구·포스터 이미지에 포함된 사회적·역사적 민감 표현을 AI(RAG 파이프라인)로 자동 진단하고, 실시간 신조어·밈 트렌드 데이터를 활용해 안전한 광고 카피를 생성·검토하는 서비스입니다.

프로그램 구성

프로그램 기술 스택 역할
program_frontend Next.js, TypeScript, Tailwind CSS 광고 문구 입력 UI, 위험도 리포트, 히스토리, 대시보드
program_backend Go 1.26, Gin, PostgreSQL 15+pgvector, OpenAI API RAG 파이프라인, REST API 서버
program_data Python 3.12, pytest, APScheduler, Selenium 신조어·뉴스 크롤링, DB 적재, 트렌드 동기화

A. 사용자 관점 기능 요구사항 (R-01 ~ R-12)

R-ID 요구사항 검증기준 시연 타임스탬프
R-01 웹에서 광고 문구를 입력하고 '검토'를 실행한다 텍스트 입력 후 '검토' 버튼 클릭 시 결과 페이지로 이동한다 01:38
R-02 문구 내 위험 표현 구간을 심각도별(high / needs_review / low)로 하이라이트한다 결과 화면에서 위험 표현이 심각도별(high·needs_review·low) 색상/밑줄로 구분 표시된다 01:45
R-03 각 위험 판정의 근거(사유·근거 출처)를 제시한다 하이라이트 항목 선택 시 판정 사유와 근거 출처가 화면에 표시된다 01:50
R-04 위험 표현에 대한 대체 문구를 1개 이상 제안한다 위험 판정 항목마다 대안 문구가 1개 이상 제시된다 02:22
R-05 입력 문구의 위험도 점수(0~100)를 산출·표시한다 결과 화면에 0~100 범위의 위험도 점수가 수치와 게이지로 표시된다 02:30
R-06 포스터 이미지를 업로드하면 OCR로 이미지 속 문구를 추출한다 이미지 업로드 후 추출된 텍스트가 화면에 표시된다 02:49
R-07 OCR로 추출한 문구를 동일한 검토 파이프라인(R-02~R-05)에 재사용한다 포스터 업로드 단독으로 하이라이트·근거·대안 문구·위험도 점수가 출력된다 02:52
R-08 트렌드어 기반으로 광고 문구를 생성하고 자동으로 리스크를 검토한다 제품·톤·트렌드 선택 시 광고 문구 후보와 각 후보의 안전 라벨·위험도가 출력된다 03:53
R-09 과거 검토 히스토리와 요약 통계를 조회한다 히스토리 목록과 총 검토 수·평균 응답시간·위험 검출율 통계 카드가 표시된다 04:14
R-10 신조어 활성도를 대시보드에서 조회한다 — 상위 활성도 신조어의 검색 트렌드 기반 활성도 추이 그래프가 표시된다 대시보드에서 신조어 활성도 순위 목록과 90일 트렌드 추이 그래프가 표시된다 04:30
R-11 민감 사건·논란 광고 사례를 대시보드에서 조회한다 대시보드에서 민감 사건 목록과 관련 광고 전례·트렌드 차트가 표시된다 04:45
R-12 웹 브라우저에서 별도 설치 없이 접근·사용할 수 있다 배포 URL 접속만으로 R-01~R-11 전 과정이 동작한다 (설치 불필요) 00:01

B. 검색·지식 내부 동작 요구사항 (R-14 ~ R-17)

R-ID 요구사항 검증기준 시연 타임스탬프
R-14 입력 문구를 임베딩해 pgvector 기반으로 의미 유사 항목을 RAG 검색한다 POST /review 요청 시 pgvector cosine 유사도 검색이 수행되고 관련 항목이 반환된다 — (내부 동작)
R-15 벡터 + 키워드(RRF) + 날짜 부스트를 융합해 근거를 랭킹한다 검색 결과가 벡터·키워드·날짜 점수를 RRF로 융합한 순서로 상위 8개가 선정된다 — (내부 동작)
R-16 과거 광고 논란·민감 이슈 시드 지식베이스를 구축한다 events·issues 테이블에 시드 데이터가 적재되어 의미 검색으로 조회된다 — (DB 적재)
R-17 정제된 지식을 임베딩해 pgvector에 저장·동기화(백필)한다 POST /knowledge/sync 호출 시 임베딩이 없는 항목에 임베딩이 생성·저장된다 — (배치 동작)

C. 비기능 요구사항 (R-18 ~ R-22)

R-ID 요구사항 검증기준 시연 타임스탬프
R-18 검토 1건 요청에 수 초 내 응답한다 — DB 적중 케이스 평균 응답시간 ≤ 3초 DB 캐시 적중 케이스 기준 평균 응답시간이 3초 이하이다 04:15
R-19 동일한 근거 집합이 주어지면 항상 동일한 위험도 점수를 산출한다 (결정론성) 동일 입력·근거 집합에 대해 반복 호출 시 항상 동일한 위험도 점수가 반환된다 — (내부 로직)
R-20 rune 오프셋 역매핑 결과가 원문 내 표현 위치와 일치한다 하이라이트 오프셋이 원문의 실제 문자(rune) 위치와 일치하며, 한글 멀티바이트 문자에서도 정확하다 — (내부 로직)
R-21 빈 입력·잘못된 요청에 4xx, 미존재 경로에 404를 반환한다 빈 body·필수 필드 누락 시 400/422, 미존재 경로 요청 시 404가 반환된다 — (오류 처리)
R-22 후보 일부 검토 실패 시 전체를 중단하지 않고 해당 슬롯만 실패로 표기한다 광고 문구 생성 중 일부 후보 검토 실패 시 해당 슬롯만 실패 처리되고 나머지 후보는 정상 반환된다 — (백엔드 동시성)
