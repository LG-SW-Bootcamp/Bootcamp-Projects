프로젝트 개요

Squeeze_PPT2Json — PowerPoint 보고서 파일(.pptx)을 자동으로 파싱하여 구조화된 JSON 및 Excel 파일로 변환하는 LLM 기반 파이프라인.

요구사항 목록

요구사항 ID 요구사항 내용 검증 기준 (완료 조건) 시연 타임스탬프
R-01 PPTX 파일을 입력 폴더(src/data/raw)에 넣으면 파이프라인이 인식하여 처리한다 src/data/raw 디렉터리 내 .pptx 파일 수와 동일한 수의 출력 파일이 src/data/raw_masked/에 생성된다 00:05
R-02 복잡한 표가 포함된 슬라이드를 자동으로 감지하고 표 영역을 플레이스홀더로 마스킹한다 Step 1-1 실행 후 입력 PPTX 각각에 대응하는 마스킹된 .pptx 파일이 src/data/raw_masked/에 생성된다 00:15
R-03 마스킹된 PPTX에서 슬라이드별 텍스트·이미지·표 구조를 JSON으로 추출한다 Step 1-2 실행 후 src/data/raw_masked_preprocessed/all_content.json이 생성되고, 각 파일 디렉터리에 extracted_content.json이 존재한다 00:30
R-04 LLM을 통해 슬라이드 내 key-value 쌍을 자동 추출한다 Step 2 실행 후 각 PPTX 파일명에 대응하는 kv_results_*.json 파일이 출력 폴더에 생성된다 00:50
R-05 추출된 key-value 결과를 LLM이 자동으로 검토하고 품질을 개선(self-refine)한다 반복 횟수(MAX_REFINE_ITERS) 설정에 따라 kv_results_*_iter{N}.json 파일이 생성되며, 초기 결과와 최종 결과 JSON이 모두 저장된다 01:15
R-06 복잡한 nested 표에서 key row·key column을 자동으로 식별하고 계층 JSON으로 변환한다 Step 3 실행 후 각 표에 대한 table.json이 results/nested_tables/{파일명}/{표자리N}/에 생성된다 01:55
R-07 본문 key-value JSON과 표 JSON을 하나의 통합 JSON으로 병합한다 Step 4 실행 후 results/merged_json/{파일명}.json이 생성되며, 표 placeholder가 실제 표 데이터로 대체된다 02:12
R-08 병합 JSON에 대해 후처리를 수행하여 최종 정제된 JSON을 생성한다 Step 4-1 실행 후 results/merged_json_post/{파일명}.json이 생성된다 02:32
R-09 최종 정제 JSON을 Excel 파일(.xlsx)로 변환한다 Step 5 실행 후 results/excel_tables/{파일명}.xlsx가 생성되고, 파일이 정상 열리며 최소 1개 이상의 행이 존재한다 02:45
R-10 웹 UI에서 PPTX 파일을 드래그&드롭 또는 파일 선택으로 업로드할 수 있다 .pptx 파일을 Dropzone에 놓거나 선택하면 업로드 요청이 전송되고 file_id가 반환된다 03:11
R-11 업로드된 PPTX의 첫 번째 슬라이드를 제출 전 미리보기로 확인할 수 있다 파일 업로드 후 SlidePreview 영역에 첫 번째 슬라이드 이미지가 표시된다 03:17
R-12 Submit 버튼을 누르면 변환이 시작되고 완료 후 JSON 결과가 화면 우측에 표시된다 Submit 클릭 시 /api/convert 요청이 전송되고, 변환 완료 후 JSON 뷰어에 결과가 렌더링된다 03:18
R-13 변환 진행 중 현재 단계를 텍스트로 실시간 표시한다 Submit 후 진행 상태 텍스트가 UI에 순차적으로 표시된다 03:18
R-14 GitHub Actions CI 파이프라인이 코드 정합성과 프론트엔드 빌드를 자동 검증한다 .github/workflows/ci.yml에 Python compile/pytest(frontend), frontend build 단계가 정의되어 main/PR에서 실행된다 N/A (CI 로그로 검증)
R-15 최종 산출물 품질 기준을 충족한다 (JSON 구조 일관성, placeholder 해소, Excel 가독성) results/merged_json_post/*.json에서 unresolved placeholder가 없고 슬라이드 구조가 dict이며, results/excel_tables/*.xlsx가 최소 1개 컬럼과 1개 이상 유효 행을 가진다 N/A (테스트 로그로 검증)

비고

  • 시연 타임스탬프는 시연영상.mp4 기준이며, 실행 환경(API 응답 속도)에 따라 ±5초 오차 발생 가능
  • R-04, R-05, R-06은 외부 LLM API 호출이 필요하므로 단위 테스트는 사전 실행 결과 파일을 기반으로 검증
  • R-10~R-13은 웹 서비스(src/program_frontend/web) 기능이며, docker compose up으로 기동 후 시연 가능
  • R-14는 시연영상 대상이 아닌 CI/CD 요구사항으로, GitHub Actions 실행 로그를 기준으로 검증
  • R-15는 결과물 품질 요구사항으로, 결과 JSON/Excel 품질 테스트 로그를 기준으로 검증
