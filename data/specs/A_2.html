SW_Bootcamp_13기 A반 2팀 — 요구사항 명세서

프로젝트 개요

  • 팀명: 보스 베이비 (A반 2팀)
  • 팀원: 정지운 · 박새란 · 김승우 · 주수원
  • 한 줄 정의: 사용자의 개인 데이터를 읽어 상황을 스스로 파악해 가전을 제어하며 정서적으로 교감하는 TV 기반 IoT 비서

핵심 차별점

  1. 컨텍스트 기반 상황 인지 — 일정·위치·결제·날씨 등 여러 개인 데이터를 종합해 현재 상황을 판단한다.
  2. 상황별 차등 동작 — 같은 “조용히 해줘”도 재택 발표 전과 퇴근 후에 다르게 동작한다.

시스템 동작 흐름

사용자 발화 → STT(노트북) → LLM API POST(노트북 → 라즈베리파이)
→ {text, homeSolution} 응답 생성(라즈베리파이)
→ TTS + 3D 캐릭터(노트북) → 가전 제어 / 결과 시각화

기술 스택

영역 구현
분산 LLM 라즈베리파이 5(8GB) × 4 · Qwen3-30B-A3B q40 · distributed-llama · OpenAI 호환 :9999 · 텐서 병렬(TP)
음성 인식(STT) Whisper
음성 합성(TTS) MeloTTS(KR) + OpenVoiceV2 음색 변환
프론트엔드 React + Three.js · 3D 캐릭터(VRM)
가전 제어 ThinQ 연동 → 시뮬레이션
개인 데이터 캘린더 · 위치 · 결제 · 날씨 · 유튜브 시청기록 등

요구사항 목록

검증 기준 내 X(임계값)는 1차 데모(6/26) 측정 후 확정합니다. 타임스탬프는 시연영상(7/1) 편집 시 실제 시점으로 갱신합니다. 현재는 영상 입증 대상 여부만 표기합니다.

요구사항 ID 요구사항 내용 검증 기준 (완료 조건) 시연 타임스탬프
R-01 사용자의 음성 발화를 텍스트로 인식한다(STT). 지정한 발화문을 말했을 때 핵심 키워드가 정확히 인식된다(또는 단어 오류율 WER ≤ X%). TBD (시연 예정)
R-02 인식 결과·상황 데이터를 로컬 라즈베리파이 LLM(OpenAI 호환 :9999)으로 POST하여 응답을 받는다. 노트북에서 POST 요청 시 HTTP 200과 응답 본문을 수신한다(통신 로그로 확인). TBD (시연 예정)
R-03 LLM 응답이 {text, homeSolution{devices}} 스키마를 따른다. 응답 JSON 파싱 시 text·homeSolution 필드가 존재하고 devices 가 허용 가전/상태로 정규화된다(스키마 검증 테스트 통과). N/A (테스트로 검증)
R-04 30B급 모델을 라즈베리파이 4대 텐서 병렬로 완전 로컬 구동한다. 4노드 클러스터가 기동되고(기동 로그) 추론 요청이 정상 처리된다. TBD (시연 예정)
R-05 응답 text를 한국어 음성으로 합성한다(TTS). 텍스트 입력 시 음성이 생성되어 재생 가능하다. TBD (시연 예정)
R-06 3D 캐릭터가 합성 음성에 맞춰 발화한다(립싱크). 음성 재생과 동시에 캐릭터의 입 모양이 동작한다. TBD (시연 예정)
R-07 응답 homeSolution.devices에 따라 가전 제어(LED/시뮬레이션)와 캐릭터 모션이 트리거된다. devices 항목에 대응하는 가전 제어(LED on/off·UI 반영)가 실행된다. TBD (시연 예정)
R-08 캘린더·위치·결제·날씨 등 복수 개인 데이터를 종합해 상황을 판단한다. 서로 다른 데이터 세트에 대해 상황 판단/응답이 달라진다(위치·결제·시청·날씨 등 복수 개인 데이터의 종합·컨텍스트 조립은 단위 테스트로 검증). TBD (시연 예정)
R-09 동일한 발화라도 컨텍스트에 따라 다르게 동작한다. “조용히 해줘”를 두 컨텍스트(재택 발표 전 / 퇴근 후)로 입력했을 때 결과가 서로 다르다(동일 발화·상이 컨텍스트 → 상이 LLM 요청/솔루션은 단위 테스트로 검증). TBD (시연 예정)
R-10 시나리오①(회식 후 퇴근): 스타일러 → 공청기(제습) → TV(드라마 이어보기) → 플레이리스트 순으로 동작한다. 시나리오① 데이터(캘린더 회식·카드 고깃집·위치 집·날씨 습도↑) 입력 시 정의된 순서대로 제어/시각화된다(스타일러→공청기→TV→플레이리스트 정의 순서는 단위 테스트로 검증). TBD (시연 예정)
R-11 시나리오②(운동 후 귀가): 냉장고 레시피 · 회복 플레이리스트(XBOOM) · TV 스트레칭 영상이 제공된다. 시나리오② 데이터(캘린더 운동·위치 헬스장) 입력 시 정의된 동작이 수행된다(냉장고 레시피·회복 플레이리스트·TV 스트레칭 정의는 단위 테스트로 검증). TBD (시연 예정)
R-12 ThinQ 연동(또는 시뮬레이션)으로 가전을 제어한다. 제어 명령 시 가전 상태가 변경되거나 시뮬레이션 UI에 반영된다. TBD (시연 예정)
R-13 가전 제어 결과를 화면에 시각화한다. 각 가전 동작별 상태/아이콘이 UI에 표시된다(devices의 name·state·status 렌더 필드 정합성은 단위 테스트로 검증). TBD (시연) + 단위 테스트
R-14 (옵션) 시청 기록(유튜브 등)을 상황 판단 컨텍스트로 활용한다. 시청 기록 데이터가 컨텍스트에 포함되어 상황 판단/응답에 반영된다(시청기록 활용은 단위 테스트로 검증). N/A (테스트로 검증, 옵션)
R-15 모델 응답 지연/처리량이 기준 이내다. 동일 조건에서 측정한 TTFT ≤ 4초 이고 생성 처리량 ≥ 10 tok/s 이며 측정 로그가 첨부된다(측정값 TTFT 2~4초·10~13 tok/s, test-results/performance.md). N/A (성능 측정 로그로 검증)
R-16 모든 추론이 로컬 LAN 내에서 처리되어 외부로 데이터가 나가지 않는다(프라이버시). 추론·음성·가전 제어 엔드포인트가 localhost/사설 LAN 만 가리킨다(단위 테스트로 검증) + 추론 중 외부 트래픽이 발생하지 않는다(네트워크 캡처 로그). N/A (단위 테스트 + 캡처 로그)
R-17 통합 UI에서 캐릭터·사용 데이터·가전 상태를 한 화면에 표시한다. 한 화면에 3개 요소(캐릭터/데이터/가전 상태)가 동시에 표시된다. TBD (시연 예정)
